# 改进需求：移动/攻击范围高亮颜色区分

> 日期：2026-04-08
> 改进点：mvRange / atkRange 高亮颜色
> 状态：待实施

---

## 目标

将移动范围（蓝色）和攻击范围（红色）用不同颜色高亮，避免玩家混淆"能移动到哪里"和"能攻击谁"。

## 现状分析

当前代码中 `mvRange()` 和 `atkRange()` 渲染函数（line 1122-1133）都使用相同的红色系高亮：

```javascript
// 当前代码
mvRange(tiles){
  c.fillStyle='rgba(220,50,50,.12)'; c.strokeStyle='rgba(220,50,50,.40)';
  ...
}
atkRange(units){
  c.fillStyle='rgba(239,68,68,.12)'; c.strokeStyle='rgba(239,68,68,.45)';
  ...
}
```

## 改动范围

### 1. `mvRange()` 渲染函数 (line 1122)
- 颜色改为蓝色系：`rgba(59,130,246,.15)` + `rgba(59,130,246,.5)`
- 与游戏整体蓝色主题一致（`--blue: #3060d0`）

### 2. `atkRange()` 渲染函数 (line 1127)
- 保持红色系（不变）

### 3. 透明度调整
- 移动范围：透明度略高（`0.15`），视觉更轻盈
- 攻击范围：透明度略低（`0.18`），提示更强烈

## 验收标准

1. 选中单位后，移动范围格子显示为**蓝色高亮**
2. 移动后，可攻击目标格子显示为**红色高亮**
3. 两者颜色对比明显，不混淆
4. 游戏整体视觉风格一致
