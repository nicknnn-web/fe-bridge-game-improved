## 1. 项目概述

一款基于浏览器的战棋游戏（Fire Emblem 风格），玩家操控 4 个我方单位（蓝方），在地图上移动、攻击 AI 控制的 4 个敌方单位（红方）。胜利条件：消灭所有敌人。

### 核心功能
- [x] 网格地图（12×12）
- [x] 回合制（玩家回合 → 敌方回合 → 循环）
- [x] BFS 路径搜索 + 移动范围高亮
- [x] 攻击范围计算 + 武器克制
- [x] 伤害计算（含暴击、反击、地形加成）
- [x] AI 敌人行动逻辑
- [x] 战斗日志
- [x] 单位详情面板
- [x] 跳过攻击按钮
- [x] 攻击动画（缩放效果）
- [x] 速度控制（1x / 0.25x / 0.1x）

|----|------|------|------|-----|-----|-----|-----|-----|------|
| p1 | 艾利乌德 | 领主 | 剑 | 28 | 10 | 5 | 5 | 5 | 1 |

| e3 | 佣兵 | 剑士 | 剑 | 22 | 12 | 4 | 9 | 8 | 1 |
| e4 | 狙击手 | 弓手 | 弓 | 18 | 9 | 2 | 6 | 5 | 2 |

### 武器克制
```
剑 > 斧 > 枪 > 剑（循环）
弓：无克制关系
```

| 攻击方武器 | 克制倍率 |
|-----------|---------|
| 剑 vs 斧 | 1.2× |
| 斧 vs 枪 | 1.2× |
| 枪 vs 剑 | 1.2× |
| 逆克制 | 0.8× |

### 伤害公式
```
damage = max(1, ATK × 武器倍率 - (DEF + 地形加成))
```

### 暴击
- 触发率：攻击方速度 - 防御方速度 ≥ 4 时，10% 概率暴击
- 暴击伤害：×2

### 反击
- 条件：防御方武器射程 ≥ 攻击方武器射程，且非暴击
- 伤害计算：同普通伤害公式

### 行动状态
- `moved`：已移动
- `acted`：已行动（攻击或跳过）
- `done = acted`（移动后可选择不攻击）
- `dead`：HP ≤ 0

### BFS 移动范围
- 以单位位置为起点
- 移动力（mv）为最大步数
- 遇障碍物（山、河流、敌方单位）停止扩展
- 桥梁可通行

## 5. 回合流程

### 玩家回合
1. 点击我方未行动单位 → 显示移动范围（蓝色格子）
2. 点击移动范围内的格子 → 单位移动 → 显示攻击范围（红色格子）
3. 点击攻击范围内的敌方单位 → 攻击动画 → 伤害结算
4. 或点击"跳过本轮攻击"按钮 → 跳过攻击
5. 所有单位行动完毕 → 自动结束回合（或手动点"结束回合"）

### 敌方回合（AI）
1. 按顺序遍历所有存活敌方单位
2. 若攻击范围内有玩家单位 → 攻击最弱者（HP 最低）
3. 若无目标 → 向最近玩家单位移动（优先攻击射程内目标）
4. 所有敌方行动完毕 → 切换回玩家回合

### 回合切换
- 玩家全员 `done` → 自动切换（无需手动点结束）
- `checkAllDone()` 检查：所有存活玩家单位 `done === true`

## 7. 动画系统

### 攻击动画
- 触发：攻击时 `attackScale = 1.5`
- 衰减：每帧 `-0.05 × speed`，降至 1.0 停止
- 实现：`canvas.save() → translate → scale → restore()`

### 伤害飘字
- 显示位置：被攻击单位格子中心
- 颜色：黄色（暴击）、红色（普通）
- 粒子爆炸效果

### 单位闪烁
- 受伤时 `flash = 5`，每帧递减
- 白色高亮闪烁效果

## 8. AI 逻辑

```
runAI():
  for each enemy:
    if 有可攻击目标:
      攻击 HP 最低目标
    else:
      BFS 寻找最佳移动位置
      评分 = 距离 - (目标血量<30% ? 10 : 0)
      移动后若在射程内则攻击
```
